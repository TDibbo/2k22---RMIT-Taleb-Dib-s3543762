My Project Idea

Weighing Needs

An iOS/android application that will integrate the touch-screen mechanic to be applied when attempting to weigh any sort of substances, this will introduce a real-time digital scaler for all users. The intended purpose for developing a real-time digital scaler that at the convenience of the user, to provide an application necessity for all weighing needs.

Create an application that allows Smartphones and Tablets to act as weighing scales. The intention behind it is to create a diverse application that requires code that is interactive with whatever substance is placed onto the device. The project will involve experimentation to deliver the indented purpose of creation, it will pursue an end of weight testing to understand what minimum threshold is required for measurement to begin.

As touch-screen devices work off electrical circuits, a mechanism will be integrated similar to how touch-screen devices operate, a compartment will be designed to be the container for weighing any type of substances. This container will be built around how touch-screen currently operates, acting as an electrical circuit to be recognised by the tablet & begin the weighing process. As this knowledge expands with the compartment, it will be deemed as a test subject to mimic the household tools available.

iPhone had created '3D Touch', this feature has been discontinued due to its complications. Though, it was an active function that people used for real-time scaling. This was introduced back in 2015, as intended the purpose of this function was to permanently change how touchscreen devices operated. As quoted "Apps and games will never feel the same again.

This technology had allowed the greater community of Apple iOS to venture into the world of 3D Touch. As per many online references reminiscing the advanced technology that Apple had to offer. The idea was removed from existence and replaced with software development to replicate the properties without requiring pressure-sensitive hardware layer. Reinventing the wheel is always used in history, and it will be reinvented

The software "Weighing Needs" will contain a user-friendly appearance & an adjustable scale dependent on the smartphone/tablet device (As there a multiple resolution sizes, it will adapt towards the size difference). It will also provide an interface that will interact with the user to adjust any settings to suit their property need. The properties involved will be responsive & functionable returning direct and precise answers. As the interface is user friendly, it will also provide notification pop-ups for first-time users. Allowing the educational stream of guidance to enable the software gets used correctly.

This will require an application coder that is proficient in 3D Touch properties entail. It will require long hours of in-depth analysis to understand weight experimenting across a broad variety of mobile devices. As experimentation is actioned on a complex playing field, it will be administered to ensure successful output responses.

As iOS & Android are two separate motherships, they require specific programming language to develop on their territory. Due to the extensive and harsh criticism that Apple's guideline presents, ensuring that all aspects are met. Further insights of what specific requirements that both platforms require.

With Apple iOS it does require proficiency in either Objective-C or Swift programming. This is not to say that an app can be developed purely on these language platforms, the Apple IDE (Integrated Development Environment) Xcode is required. Xcode IDE is a graphical interface that provides a complete set of tools for application creation on iOS platform. The iOS SDK (Software Development Kit) also includes a variety of APIs and frameworks providing access to additional features to enhance the application uses & implement features such as; Siri, Notifications, Maps, Touch ID, social networks etc. Last but not least, you must be a member of the Apple Developer Program. This membership simply provides more tools & resources.

The main programming languages that are used for Android devices are Visual C #, C ++ and Java. The most preferred language used is Java due to its simplicity of operating an application. The breakdown of application creation on Android is; Java is referred to being the logic of the program, XML is the representation of the program. After these two key points are finalised, then acquiring the assembly tool to compile all the files and package them together into an .apk file.
